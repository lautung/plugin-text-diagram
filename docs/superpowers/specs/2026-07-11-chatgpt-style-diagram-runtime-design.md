# ChatGPT 风格文本绘图前台运行时设计

## 背景

当前插件已经具备 Mermaid 前台渲染能力，并在开发分支中尝试加入 ChatGPT 风格预览卡片。但前台可能在插件设置缺失、运行时资源未正确加载或选择器未生效时只显示 Mermaid 源码。与此同时，卡片样式和交互主要以大段 Java 文本块存在于 `JSInjector` 中，不利于继续加入 PlantUML、主题切换、响应式策略和完整交互。

本设计将前台渲染迁移为独立的 TypeScript/CSS 运行时，使 Mermaid 和 PlantUML 在文章与单页中使用统一的 ChatGPT 风格卡片。

## 目标

- 修复前台只显示 Mermaid 源码而未渲染的问题。
- Mermaid 与 PlantUML 统一使用 ChatGPT 风格卡片。
- 工具栏提供源码/预览切换、全屏、复制源码和下载。
- 使用接近 ChatGPT 的细线 SVG 图标，并保持统一尺寸、线宽和状态反馈。
- 自动跟随 Halo 前台明暗主题，并响应运行时主题变化。
- 在 Halo 插件后台提供三种窄屏策略，默认使用横向滚动。
- 保持首次加载、PJAX、重复渲染过滤、字体等待和 `htmlLabels: false` 等现有兼容行为。
- 让任一图表的失败不影响正文和其他图表。

## 非目标

- 不改变编辑器中图表节点的存储格式。
- 不引入新的图表语言。
- 不改变 PlantUML 服务端生成图片的现有方式。
- 不手工修改 `src/main/resources/ui`、`src/main/resources/static` 或其他构建生成内容。
- 不为窄屏断点增加单独的管理员配置；断点固定为 `768px`。

## 总体架构

采用独立前台运行时方案。Java 后端只负责读取设置、安全序列化配置，以及向文章和单页注入运行时 CSS/JS 资源。前台 TypeScript 运行时负责节点扫描、渲染调度、统一卡片、主题响应和窄屏行为。

源代码边界如下：

```text
console/src/runtime/text-diagram/
├── index.ts
├── card.ts
├── types.ts
├── mermaid-adapter.ts
├── plantuml-adapter.ts
├── responsive.ts
├── theme.ts
└── style.css
```

- `index.ts`：运行时入口，读取配置、扫描节点、处理首次加载和 `pjax:success`。
- `card.ts`：创建统一卡片并管理工具栏、源码视图、全屏、复制和下载。
- `types.ts`：定义运行时配置、图表适配器、渲染结果和错误结果接口。
- `mermaid-adapter.ts`：加载并初始化 Mermaid，等待字体，渲染 SVG，转换语法及加载错误。
- `plantuml-adapter.ts`：读取现有图片地址与源码，管理图片加载、下载信息和失败状态。
- `responsive.ts`：应用三种窄屏策略和固定 `768px` 断点。
- `theme.ts`：检测明暗模式并监听主题变化。
- `style.css`：卡片、工具栏、画布、源码、错误、全屏、暗色及窄屏样式。

Rsbuild 负责把运行时入口和样式生成到插件公开资源目录。Java 代码不得承载完整前台实现，也不得手工维护生成后的资源文件。

## 后端职责与设置

文章和单页内容处理器保持成对行为：

1. 从设置组 `basic` 读取配置。
2. 对缺失、空字符串或部分缺失的设置应用默认值。
3. 安全序列化配置，避免选择器或管理员输入破坏脚本上下文。
4. 注入运行时 CSS/JS 引用和当前页面配置。
5. 发生设置读取或注入错误时记录错误并保留原始正文。

保留现有设置字段：

- `dark_class_selector`
- `mermaid_selector`

新增窄屏策略字段，取值为：

- `scroll`：横向滚动，默认值。
- `scale`：自动缩放图表以适应卡片宽度。
- `thumbnail`：正文显示缩略图，点击后进入全屏查看。

该策略只在视口宽度小于 `768px` 时生效；桌面端保持正常展示。

## 节点扫描与渲染数据流

1. 运行时读取 Java 注入的页面配置。
2. 扫描 `text-diagram[data-type]` 节点。
3. 初始化前保存原始源码、图表类型和必要资源地址。
4. 使用 `data-text-diagram-state` 状态属性标记节点，防止重复包装和重复绑定。
5. 根据 `data-type` 选择适配器：
   - `mermaid` 交给 Mermaid 适配器。
   - `plantuml` 交给 PlantUML 适配器。
   - 未识别类型保持原始内容，不创建错误卡片。
6. 渲染成功后，把适配器结果放入统一卡片画布。
7. 渲染失败时保留卡片，画布显示简洁错误信息，同时保留源码、复制和全屏能力。
8. `pjax:success` 仅扫描新增或未处理节点；已处理节点不重复包装、监听或渲染。

Mermaid 渲染继续满足：

- 异步、单例加载 Mermaid 资源。
- 避免并发初始化和过期结果覆盖。
- 渲染前等待字体。
- `startOnLoad: false`。
- `htmlLabels: false`。
- 流程图设置 `useMaxWidth: false`，由卡片响应式策略控制展示。

PlantUML 继续使用编辑器生成并保存的图片地址。适配器保留对应源码，使源码视图、复制和错误降级与 Mermaid 保持一致。

## 卡片与交互设计

Mermaid 和 PlantUML 共享同一套卡片结构：

- 顶部工具栏左侧显示图表类型及图标。
- 工具栏右侧依次提供源码、预览、全屏、复制和下载。
- 主体区域显示渲染结果或错误提示。
- 源码区域使用等宽字体并允许横向滚动。

图标采用内联细线 SVG，不依赖操作系统字体：

- 统一使用约 `18px` 尺寸和一致线宽。
- 默认透明背景，悬停时显示浅灰背景。
- 当前源码或预览状态使用浅灰圆形背景。
- 每个按钮提供 `aria-label`、`title` 和键盘聚焦轮廓。
- 复制成功后短暂切换为勾选状态。
- 不可用的下载按钮降低透明度并禁止交互。

下载规则：

- Mermaid 下载 SVG。
- PlantUML 下载当前渲染图片，并保留由资源内容或地址确定的正确扩展名。

全屏规则：

- 卡片覆盖视口并保留源码/预览切换。
- 支持按 `Escape` 退出。
- `thumbnail` 窄屏策略点击缩略图时进入相同的全屏模式。

## 主题与响应式

主题由 `dark_class_selector` 检测。运行时监听相关 DOM 变化并更新卡片主题。主题变化时只更新必要状态；Mermaid 仅在主题确实影响图表配色时重新渲染，不重新创建整个运行时或重复绑定事件。

窄屏策略在小于 `768px` 时生效：

- `scroll`：卡片保持正文宽度，图表画布横向滚动；工具栏允许收紧或换行。
- `scale`：根据可用宽度缩放整张图，避免横向滚动，但允许文字变小。
- `thumbnail`：正文中展示缩略图，点击后以全屏方式查看完整图表。

宽屏设备不受上述设置影响，默认正常展示并允许宽图画布横向滚动作为安全降级。

## 错误处理与清理

- 设置缺失或字段为空：使用内置默认值并继续加载运行时。
- 配置序列化失败：记录错误并保留原始正文。
- Mermaid 加载失败或超时：当前卡片显示加载错误，保留源码、复制和全屏能力；清理失败的加载状态，使后续扫描可以重试。
- Mermaid 语法错误：只标记当前图表，不阻断同页其他图表。
- PlantUML 图片加载失败：显示图片加载错误，保留源码视图，禁用下载。
- 剪贴板失败：切换到源码视图并选中内容，便于手动复制。
- 节点移除或运行时清理：释放临时对象 URL，避免遗留无效下载资源。
- 所有详细错误记录到浏览器控制台，但前台只显示简洁、可理解的信息。

## 测试与验收

### Java 单元测试

- 设置为空、缺失和部分字段缺失时应用正确默认值。
- 配置字符串安全转义或安全序列化。
- 文章与单页处理器注入相同资源和配置。
- CSS/JS 资源只注入一次，公开路径正确。
- 后端异常时保留原始正文。

### TypeScript 单元测试

- 节点扫描、类型分发和重复节点过滤。
- Mermaid 与 PlantUML 适配器的成功和失败结果。
- 源码/预览切换、复制、下载、全屏和 `Escape`。
- 复制失败时的源码选择降级。
- 明暗主题运行时切换。
- 三种窄屏策略、默认 `scroll` 和 `768px` 生效边界。
- PJAX 新节点渲染且旧节点不重复绑定。
- 清理时释放对象 URL。

### Chrome 人工验收

- Mermaid：合法语法、非法语法、宽图、多图及快速 PJAX 切换。
- PlantUML：正常图片、图片失败、源码切换和下载。
- 浅色与深色主题运行时切换。
- 桌面端及小于 `768px` 时的三种后台策略。
- 工具栏图标、键盘焦点、全屏、复制和真实下载行为。
- 一张图失败时，其余正文和图表仍正常工作。

## 完成标准

- 文章和单页中的 Mermaid 与 PlantUML 均显示统一的 ChatGPT 风格卡片。
- 当前仅显示源码的 Mermaid 场景恢复正常渲染。
- 后台可选择三种窄屏策略，未配置时默认横向滚动。
- 完整工具栏、自动主题、错误降级和 PJAX 行为通过自动化与 Chrome 验收。
- 前台运行时逻辑从 Java 大文本块迁移到可类型检查、可独立测试的 TypeScript/CSS 源码。
- 所有构建产物由现有构建流程生成，不手工编辑或提交被仓库排除的生成目录。
